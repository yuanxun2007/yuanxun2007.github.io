Object-C中的布尔类型(BOOL)

布尔类型用来声明存储真值和假值的变量。在Object-C中，类型BOOL,具有YES和NO两个值。

1. 首先来看看定义，在objc.h(ios sdk 8.4及osx sdk 10.10)文件中，我们看到如下定义：

#define OBJC_BOOL_DEFINED

/// Type to represent a boolean value.
#if !defined(OBJC_HIDE_64) && TARGET_OS_IPHONE && __LP64__
typedef bool BOOL;
#else
typedef signed char BOOL; 
// BOOL is explicitly signed so @encode(BOOL) == "c" rather than "C" 
// even if -funsigned-char is used.
#endif

#if __has_feature(objc_bool)
#define YES __objc_yes
#define NO  __objc_no
#else
#define YES ((BOOL)1)
#define NO  ((BOOL)0)
#endif

从上面的文件中，可以梳理出2点：

  • BOOL的定义，在64位的ios和32位ios、mac os，定义是不一样的。在64位ios中，它就是一个bool类型;在32位ios、mac os中，则是一个带符号的字符类型。
  • BOOL的值也有2种情况，在64位ios，就是YES和NO, 在32位ios、mac os，1表示YES，其他表示NO

2. 再来看一下使用

从上面我们知道，BOOL只有在64位ios中才是真正的布尔类型值，在32位ios及mac os中，编译器将BOOL认做8位二进制数，YES和NO值只是一个约定。所以使用的时候就需要注意以下2点：

  • 在32位ios及mac os中，不要直接的跟YES进行比较判断真假，这么做的话，可能会得到意想不到的结果
  • 在32位ios及mac os中，不小心的将长于1字节整型值赋给了BOOL变量，就只有低位会起作用（例如8960，写成16进制，就是0x2300,赋给BOOL变量，BOOL值将会是0，就是NO值）。

我建了2个工程进行测试验证，执行下面的代码及结果如下：

关于Object-C中@用法的梳理

@是Object-C的一个特性，可以认为是对C的扩展，梳理了一下，大概有以下几个用法：

1. 用的最多的，就是字面量，语法很精简。把字符、数值封装到Object-C的对象中，同时也可以用来创建数组，字典。例如：

1. 封装字符串： NSString *someString = @"Hello World!";
2. 封装整型： NSNumber *intNumber = @1;
3. 封装浮点型数值： NSNumber *floatNumber = @1.06f;
4. 封装布尔型数值： NSNumber *boolNumber = @YES; 
5. 创建数组：NSArray *animals = @[@"dog", @"cat", @"bird", @"fish"];
6. 创建字典：NSDictionary *contactDictionary = @{ @"firstName":@"Carl", 
                                                @"last Name":@"Fu", 
                                                @"mobile":@"xx", 
                                                @"mail":@"xx@xx.com", nil};

2. 定义一条指令。比如：

1. 定义一个类：@interface Shape : NSObject;
2. 定义一个实现：@implemention Shape;
3. 定义一个接口：@protocol UIWebViewDelegate <NSObject>;

3. 输出Object-C对象。比如：

NSLog(@"The number is %@", someNumber);

#include #import @class 详解及区别

1. #include

[1] #include :引用系统自带的头文件，编译器会在系统目录下查找.

[2] #include “x.h”:引用用户自定义的文件，编译器首先会在用户目录下查找，然后到安装目录下查找，最后在系统文件中查找.

[3] 使用#include的时候要注意重复引用，一般使用#ifndef命令来解决，列如：

#ifndef _CLASSA_H

#define _CLASSA_H

#include "ClassA"

#endif

2. #import

Object-C使用头文件来包含元素声明，这些元素包括结构体、符号常量、函数原型等。Object-C语言使用#import语句来通知编译器在头文件中查询定义，功能和#include基本一致，不过#import可保证头文件只被包含一次，而不论此命令实际上在那个文件中出现了多少次。

3. @class

声明一个类，告诉编译器它后面是一个类的名字，不用管实现。在实现文件，如果需要使用这个类，则需要#import这个类，才会把类的引用信息包含进来。

4. #import、#include二者的区别

前者不会出现引用重复引用，保证在同一个文件中，一个头文件只会被#import一次；而后者如果不注意，则有可能导致重复引用同一个头文件。

5. #include、#import、@class三者的区别

#include和#import引用类的时候，会包含类的所有信息，包括变量方法等，会对编译效率产生一定的影响，而@class则仅仅是告诉编译器一个类的声明，不会把类的引用信息包含进来。所以一般在@interface中，引用一个类用@class就好了。

什么是cocoa?

在面试中被问到，感觉一脸茫然，不知道怎么答起？只是隐约记得好像是foundation和库之类的一些东西。说明我的知识掌握确实不是太好。回到家中补了一下课，梳理了一下，大概有以下几点吧：

1. cocoa的前生：

1985年， steve jobs创建了NeXT公司,致力于开发强大且经济的工作站。NeXT选择Unix作为其操作系统，创建了NextSTEP。NextSTEP只是创造了一些特性，拥有少量忠实用户，并未在商业上获得成功。

1996年，苹果公司收购NeXT之后，NextSTEP被重命名为cocoa，并得到了Macintosh编程人员的广泛认可。

2. cocoa的是什么：

使用Object-C开发的一款强大的用户界面工具包。

3. cocoa的现在：

经过多年的发展，Cocoa已经演化成一个功能强大的优秀工具集。
