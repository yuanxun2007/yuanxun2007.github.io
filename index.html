Object-C继承

继承是OOP的一个重要特性，处理类与对象间的关系。

1.OOP处理类与对象

  • 继承：子类继承父类，可以继承父类的方法和属性，实现对象的多态和代码重用。
  • 复合：对象引用其他对象时，可以利用其他对象提供的特性。

2.为何要使用继承

继承的出现主要是为了解决现实中，很多对象有共同的特性，如果每个对象都去建立一套的话，就会出现很多重复的代码，而且在维护的时候也需要多套代码，很明显这是不可取的。所以继承就是为了解决代码的重用性和可维护性。

3.继承的语法

@interface Circle : NSObject， 在@interface后面是定义的类名， 冒号后面是继承的类名。而且Object-C中，只支持单重继承。

4.继承的工作机制

  • 1.继承的方法调度：是通过Object-C的消息机制实现的，当代码发送消息的时候，Object-C的方法调度程序首先会在当前的类中搜索相应的方法；如果在当前类中找不到，就会将消息转发到该对象的父类中查找；如果仍然没有找到，则会在未定义的方法中查找；如果仍然没有找到，则会抛出异常。
  • 2.子类调用父类：使用super关键字，编译器会识别这个关键字，向父类发送一个消息，父类接受消息之后，仍然按照方法调度的机制，查找是否能够响应，如果不能则转发到父类查找。
  • 3.重写/覆盖父类方法：直接在当前类中，实现父类一样方法名的方法，以实现替换或者增强由超类定义的方法。

Object-C面向对象

面向对象编程(Object-Oriented Programming)的首字母缩写为OOP，这是一种编程技术，最初是为编写模拟程序而开发的。

1. 首先要了解一下使用间接

间接的含义很简单——不在代码中直接使用某个值，而是使用该值的指针。在Object-C里面，见的最多的就是指针，引起问题的，绝大部分也是指针。不管是面向过程还是面向对象，都使用了这个技术。

2. 面向过程 & 面向对象

  • 面向过程：分析出解决问题所需的步骤，然后用函数把这些步骤一步一步实现，使用的时候一个一个依次调用就可以了。偏向于机器的思考方式
  • 面向对象：把构成问题事务分解成各个对象，建立对象的目的不是为了完成一个步骤，而是为了描述某个事物在整个解决问题的步骤中的行为。偏向于人类的思考方式

3. 产生面向对象的历史原因

  • 计算机是帮助人类解决问题的，然而终究只是一个机器，只会按照程序员写的代码一步一步执行得到结果。结构化编程，就是按照机器的思维写出的代码，如果系统很大，逻辑很复杂，那么后期的维护和扩展将非常困难。
  • 结构化设计是以功能为目标来设计构造应用系统，这种做法导致我们程序设计时，不得不将客体所构成的现实世界映射到由功能模块组成的解空间中，这种空间转换，背离了人们观察和解决问题的思路。

4. 面向对象所解决的问题

面向对象是在面向过程之后发展起来的，主要是为了解决人们追求的系统可维护性、扩展性、重用性问题。

5. 面向对象的基本概念

  • 对象：问题域中客观存在的实体和主观抽象的概念，即是对象
  • 类：对象的模版，是在对象之上的抽象。类与对象的关系，类似于数据类型与变量的关系。
  • 消息：对象之间通信的描述。一般由三部分组成：接收消息的对象、消息名称、传递的变元。

6. 面向对象的三大特性

  • 封装：可以认为是对象的说明或者描述。封装了对象的数据和方法。通过不同的权限控制其他对象对它的访问。封装把对象的变化封装起来，在不影响其他部分的情况下修改或者扩展变化的部分，这是所有设计模式的基础。因此封装的作用，就解决了程序的可扩展性。
  • 继承：子类继承父类，可以继承父类的方法和属性，实现了多态和代码重用。因此也解决了系统的重用性和可扩展性。另外一方面，继承也一定程度上破坏了封装性，因为他是对子类开放的，修改父类会影响所有的子类，所以要慎用继承，设计时要尽量遵守开闭原则，优先考虑使用组合，而不是继承，这是面向对象开发的一个重要经验。
  • 多态：对象根据所接收的消息而做出的动作，不同的对象接收消息时做出的动作可能完全不同，这种现象称为多态。在程序中，接口是动作的抽象，接口的主要目的是为不相关的类提供通用的处理服务，多态就是接口的多种不同实现方式。比如鸟会飞，但是超人也会飞，通过飞这个接口，我们可以让鸟和超人，都实现这个接口，这就实现了系统的可维护性，可扩展性。

7. 在Object-C中如何体现面向对象

  • 类：使用@interface声明类，在@interface中定义类包含的数据与方法,实现类的封装。
  • 协议：使用@protocol声明协议，定义通用的方法。然后通过在不同类中@implementation实现同一个方法，不同的操作，实现了类的多态。
  • 继承：在@interface声明的类名后面，有一个:NSObject，即代表继承自NSObject类。在Object-C中，只支持单重继承，所有类都继承自NSObject或者NSProxy这2个基类，而且这2个基类都实现了协议NSObject。子类中可以使用super调用父类，使用跟父类一样的方法名，覆盖父类方法，在自己的方法中，做一些扩展，调用父类的方法，即可重载父类的方法。
  • 消息：Object-C的消息发送机制，转发机制和响应机制，描述了对象之间的通信。其实这个消息机制是学习Object-C的一个核心，理解了这个机制对于开发、分析问题是很有帮助的。

结尾语

上面的这些，都是在网上摘抄的，以方便自己理解这一块内容，也算是一个总结吧。对于Object-C里面，具体到每一个特性，都可以写一篇文章出来，这里算是大概的理解一下面向对象是个什么东西，Object-C中怎么对应面向对象这些特性吧。

__LP64__数据模型

最近看iOS sdk，发现标志LP64，不知道什么意思，赶紧脑补了一下，从网上摘了点知识过来。

1. 基础数据类型模型定义

C/C++仅仅定义基本数据类型的关系（字长：CHAR<=SHORT<=INT<=LONG），并没有严格定义它们的字长，根据编译器的不同实现，它们的字长表示也不同

  • LP64指的是LONG/POINTER字长为64位；
  • ILP64指的是INT/LONG/POINT字长为64位；
  • LLP64指的是LONGLONG/POINTER字长为64位；
  • ILP32指的是INT/LONG/POINTER字长为32位；
  • LP32指的是LONG/POINT字长是32位的，INT字长为16位。

2. 数据模型中数据类型的长度表

ILP32和LP32是32位平台上字长的数据模型，LP64、ILP64、LLP64是64位平台上字长的数据模型

TYPE           LP32    ILP32    LP64    ILP64    LLP64

CHAR           8        8        8        8        8

SHORT          16       16       16       16       16

INT            16       32       32       64       32

LONG           32       32       64       64       32

LONG LONG      64       64       64       64       64

POINTER        32       32       64       64       64

3. 一些注意点

  • 32位Windows采用的是ILP32数据模型，64位Windows采用LLP64数据模型。
  • 32位的Linux/Unix使用ILP32数据模型，64位Linux/Unix使用LP64数据模型。
  • 为了增加代码的移植性，打印无符号整形数，不管申明时是如何定义的，统一使用 %lu。
  • 为了保证平台的通用性，代码中尽量不要使用long数据库型。
  • 使用INT时也可以使用intptr_t来保证平台的通用性，它在不同的平台上编译时长度不同，但都是标准的平台长度，比如：64位机器上长度为8字节，32位机器上长度为4字节。
  • 编写代码时要尽量使用sizeof来计算数据类型的大小。
  • ssize_t和size_t分别是signsize_t和unsigned signed size of computer word size。它们也是表示计算机的字长，在32位机器上是int型，在64位机器上long型，从某种意义上来说它们等同于intptr_t和 uintptr_t。

Object-C中的布尔类型(BOOL)

布尔类型用来声明存储真值和假值的变量。在Object-C中，类型BOOL,具有YES和NO两个值。

1. 首先来看看定义，在objc.h(ios sdk 8.4及osx sdk 10.10)文件中，我们看到如下定义：

#define OBJC_BOOL_DEFINED

/// Type to represent a boolean value.
#if !defined(OBJC_HIDE_64) && TARGET_OS_IPHONE && __LP64__
typedef bool BOOL;
#else
typedef signed char BOOL; 
// BOOL is explicitly signed so @encode(BOOL) == "c" rather than "C" 
// even if -funsigned-char is used.
#endif

#if __has_feature(objc_bool)
#define YES __objc_yes
#define NO  __objc_no
#else
#define YES ((BOOL)1)
#define NO  ((BOOL)0)
#endif

从上面的文件中，可以梳理出2点：

  • BOOL的定义，在64位的ios和32位ios、mac os，定义是不一样的。在64位ios中，它就是一个bool类型;在32位ios、mac os中，则是一个8位带符号的字符类型。
  • BOOL的值也有2种情况，在64位ios，就是YES和NO, 在32位ios、mac os，1表示YES，其他表示NO

2. 再来看一下使用

从上面我们知道，BOOL只有在64位ios中才是真正的布尔类型值，在32位ios及mac os中，编译器将BOOL认做8位二进制数，YES和NO值只是一个约定。所以使用的时候就需要注意以下2点：

  • 在32位ios及mac os中，非零值未必是BOOL的YES，但是0 一定是NO。所以最好不要直接的跟YES进行比较判断真假，这么做的话，可能会得到意想不到的结果
  • 在32位ios及mac os中，不小心的将长于1字节整型值赋给了BOOL变量，就只有低位会起作用（例如8960，写成16进制，就是0x2300,赋给BOOL变量，BOOL值将会是0，就是NO值）。

关于Object-C中@用法的梳理

@是Object-C的一个特性，可以认为是对C的扩展，梳理了一下，大概有以下几个用法：

1. 用的最多的，就是字面量，语法很精简。把字符、数值封装到Object-C的对象中，同时也可以用来创建数组，字典。例如：

1. 封装字符串： NSString *someString = @"Hello World!";
2. 封装整型： NSNumber *intNumber = @1;
3. 封装浮点型数值： NSNumber *floatNumber = @1.06f;
4. 封装布尔型数值： NSNumber *boolNumber = @YES; 
5. 创建数组：NSArray *animals = @[@"dog", @"cat", @"bird", @"fish"];
6. 创建字典：NSDictionary *contactDictionary = @{ @"firstName":@"Carl", 
                                                @"last Name":@"Fu", 
                                                @"mobile":@"xx", 
                                                @"mail":@"xx@xx.com", nil};

2. 定义一条指令。比如：

1. 定义一个类：@interface Shape : NSObject;
2. 定义一个实现：@implemention Shape;
3. 定义一个接口：@protocol UIWebViewDelegate <NSObject>;

3. 输出Object-C对象。比如：

NSLog(@"The number is %@", someNumber);

#include #import @class 详解及区别

1. #include

[1] #include :引用系统自带的头文件，编译器会在系统目录下查找.

[2] #include “x.h”:引用用户自定义的文件，编译器首先会在用户目录下查找，然后到安装目录下查找，最后在系统文件中查找.

[3] 使用#include的时候要注意重复引用，一般使用#ifndef命令来解决，列如：

#ifndef _CLASSA_H

#define _CLASSA_H

#include "ClassA"

#endif

2. #import

Object-C使用头文件来包含元素声明，这些元素包括结构体、符号常量、函数原型等。Object-C语言使用#import语句来通知编译器在头文件中查询定义，功能和#include基本一致，不过#import可保证头文件只被包含一次，而不论此命令实际上在那个文件中出现了多少次。

3. @class

声明一个类，告诉编译器它后面是一个类的名字，不用管实现。在实现文件，如果需要使用这个类，则需要#import这个类，才会把类的引用信息包含进来。

4. #import、#include二者的区别

前者不会出现引用重复引用，保证在同一个文件中，一个头文件只会被#import一次；而后者如果不注意，则有可能导致重复引用同一个头文件。

5. #include、#import、@class三者的区别

#include和#import引用类的时候，会包含类的所有信息，包括变量方法等，会对编译效率产生一定的影响，而@class则仅仅是告诉编译器一个类的声明，不会把类的引用信息包含进来。所以一般在@interface中，引用一个类用@class就好了。

什么是cocoa?

在面试中被问到，感觉一脸茫然，不知道怎么答起？只是隐约记得好像是foundation和库之类的一些东西。说明我的知识掌握确实不是太好。回到家中补了一下课，梳理了一下，大概有以下几点吧：

1. cocoa的前生：

1985年， steve jobs创建了NeXT公司,致力于开发强大且经济的工作站。NeXT选择Unix作为其操作系统，创建了NextSTEP。NextSTEP只是创造了一些特性，拥有少量忠实用户，并未在商业上获得成功。

1996年，苹果公司收购NeXT之后，NextSTEP被重命名为cocoa，并得到了Macintosh编程人员的广泛认可。

2. cocoa是什么：

使用Object-C开发的一款强大的用户界面工具包。

3. cocoa的现在：

经过多年的发展，Cocoa已经演化成一个功能强大的优秀工具集。
